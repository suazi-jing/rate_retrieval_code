## 🏗️ 系统架构

系统分为三大模块：

- **数据采集模块**：Xposed Hook 实现汇率实时抓取，数据清洗入库
- **智能计算模块**：Dify 工作流引擎，意图识别 + 知识库 + LLM 生成 + SQL 查询
- **消息触达模块**：WhatsApp Webhook + wasender API 联调，多轮对话记忆
---

## 🚀 功能模块

### 1. 数据采集模块

针对封闭 IM 平台的汇率数据，采用 **Android 模拟器 + Xposed Hook** 非侵入式采集方案：

- 通过运行时 Hook 框架拦截目标应用消息事件
- 实现汇率广告的实时捕获与文本化
- 数据自动清洗、结构化入库 MySQL

IM平台消息 → Xposed Hook 拦截 → 数据清洗 → MySQL 入库


### 2. 数据拆分入库模块

原始汇率广告包含大量行业术语，需要结构化处理后才能使用：

- **lx_db_insert.py**：从 MySQL 抽取原始汇率广告，远程调用 Dify Workflow API
- **Dify 工作流**：LLM 节点将广告拆分为 XML 格式的结构化数据
- **bd_insert_api.py**：接收 Dify HTTP 请求，将结构化数据入库，并删除已处理的原始记录
原始广告数据 → Python 抽取 → Dify LLM 拆分 → XML 结构化 → API 入库 → 删除原始数据


### 3. 智能报价模块（Dify 工作流）

**webhook_server.py** 部署在服务器上，调用 wasender API 监听 WhatsApp 消息，捕获后回调 Dify 工作流：

#### 工作流程
**步骤①**：意图分类（问价/闲聊），根据不同信息走不同流程

**步骤②**：知识库检索，包括 12 种卡类信息询问规则和外汇值查询

**步骤③**：LLM 根据知识库中的卡类询问规则和用户问题，生成对应的 JSON 文本

**步骤④**：JSON 数据交给代码执行组合查询，生成卡标签（卡名、国家、卡类型）和 SQL 查询语句

**步骤⑤**：数据库查询 + 外汇值 + 卡标签 + 用户初始信息

**步骤⑥**：综合计算报价，通过 HTTP 请求访问 wasender API 发送到用户 WhatsApp


#### 核心能力

- **意图识别**：自动区分问价信息和闲聊信息，分流处理
- **知识库检索**：内置 12 种礼品卡询问规则和实时外汇值查询
- **LLM 结构化生成**：根据知识库规则和用户问题，生成 JSON 格式查询参数
- **动态 SQL 查询**：代码节点组合卡标签（卡名 + 国家 + 卡类型），拼接 SQL 语句查询数据库
- **自动报价计算**：综合 LLM JSON 数据、知识库外汇值、数据库查询结果，自动计算最终报价
- **多轮对话记忆**：通过 API 调用 Dify 实现会话记忆功能，支持上下文连续对话
